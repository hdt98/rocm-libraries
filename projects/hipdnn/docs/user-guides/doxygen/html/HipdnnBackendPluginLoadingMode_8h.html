Detailed Description

Plugin loading mode configuration for hipDNN.

This file defines the plugin loading modes that control how hipDNN loads and manages execution engine plugins. Plugins provide the actual GPU kernel implementations for deep learning operations.

enum hipdnnPluginLoadingMode_ext_t

Specifies the plugin loading mode for hipDNN (extension API)

This enumeration defines how plugins are loaded into hipDNN. The loading mode affects which plugins are available for engine discovery and execution.

Enumerator
HIPDNN_PLUGIN_LOADING_ADDITIVE 

Additive loading mode (default)

Loads all user-specified plugins in addition to the default plugins. This is useful when you want to extend hipDNN with custom plugins while retaining access to the built-in implementations.

HIPDNN_PLUGIN_LOADING_ABSOLUTE 

Absolute loading mode.

Loads only the user-specified plugin from the most recent registration call, ignoring all default plugins. This is useful when you want complete control over which plugins are active.
