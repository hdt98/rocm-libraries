## Introduction

CK Profiler is a tool that we use for benchmarking the CK Tile grouped convolution implementations. 
The CK profiler can support both the old CK and new CK Tile kernels. 
In this project, we are interested in the CK Tile convolutions. 

Currently the CK Tile convolutions are integrated in the CK profiler using the CK Builder 
(sse the CK Builder [Redame](experimental/builder/README.md)) to generate the instances from [configuration files](experimental/grouped_convolution_tile_instances/configs). 
The CK Tile convolution instances are defined in directory `experimental/grouped_convolution_tile_instances` 
and the design is described in the [Readme](experimental/grouped_convolution_tile_instances/README.md). 
The code generation (C++ code from config files) is done via python script `experimental/grouped_convolution_tile_instances/generate_instances.py`.

The CK Tile instances generated by the CK Builder are injected to the CK profiler via three header files
- profiler/include/profiler/grouped_convolution_forward_tile_algs.hpp (FWD)
- profiler/include/profiler/grouped_convolution_backward_data_tile_algs.hpp (BWD data)
- profiler/include/profiler/grouped_convolution_backward_weight_tile_algs.hpp (BWD weight)

In this project we want to replace the CK Builder based code generation with the code generation provided by the [CK Dispatcher](dispatcher/README.md).

## Building

To configure the CMake project, run 

```bash
cmake                                                                                             \
  -D CMAKE_PREFIX_PATH=/opt/rocm                                                                  \
  -D CMAKE_CXX_COMPILER=/opt/rocm/bin/hipcc                                                       \
  -D CMAKE_BUILD_TYPE=Release                                                                     \
  -D GPU_TARGETS="gfx950"                                                                         \
  -D CK_EXPERIMENTAL_BUILDER=ON                                                                   \
  -D CMAKE_CXX_STANDARD=20                                                                        \
  -G Ninja                                                                                        \
  ..
```

in directiory `build-gfx950`.

To build the CK profiler, run command 
```
ninja -j128 ckProfiler
```
in the build directory `build-gfx950`. 

When we move to the dispatcher based code generation, you need to add the dispatcher flags to CMake configure step.

## Project plan

We will do the porting in three stages

1. Switch bwd weigth to use dispatcher
2. Switch fwd to use dispatcher
3. Switch bwd data to use dispatcher

The bwd weight is the most complex and once we have it working, we can proceed with steps 2. and 3.
