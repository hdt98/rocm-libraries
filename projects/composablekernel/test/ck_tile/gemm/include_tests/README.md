# Self-Contained Include Tests for CK_Tile GEMM Headers

This directory contains automatically generated test files to verify that all header files in `include/ck_tile/ops/gemm/` are self-contained (i.e., can be compiled independently without missing includes).

## Purpose

These tests ensure that each header file includes all necessary dependencies and doesn't rely on transitive includes from other headers that might include it. This is a best practice for library development that:

- Prevents fragile dependencies on include order
- Makes headers more maintainable
- Catches missing `#include` directives early
- Ensures headers can be used independently

## Structure

Each test file:
1. Includes exactly ONE header from `include/ck_tile/ops/gemm/`
2. Contains a minimal `main()` function that does nothing
3. If it compiles successfully, the header is self-contained

## Generation

These tests are generated by the `generate_include_tests.sh` script in the parent directory.

To regenerate the tests (e.g., after adding new headers):

```bash
cd example/ck_tile/03_gemm
./generate_include_tests.sh
```

## Building

The tests are automatically built when you build the `example/ck_tile/03_gemm` directory with appropriate GPU targets:

```bash
cd build
cmake -DGPU_TARGETS="gfx90a" ..
make -j$(nproc)
```

All test executables will be prefixed with `tile_test_include_*`.

## Troubleshooting

If a test fails to compile:
1. The header is missing an `#include` directive
2. Add the missing include to the header file
3. Rebuild to verify the fix

Example error:
```
error: 'SomeType' was not declared in this scope
```

Fix: Add `#include "path/to/header_defining_SomeType.hpp"` to the failing header.
