# Launchers

Current DeepSeek-V4 target launcher:

- `torchtitan_direct_deepseek_v4_4x8xmi350_canary.sh`: direct TorchTitan `torchrun` path for the real DeepSeek-V4-Flash canary family. The default config is now the active `flash_12layer_ceiling_probe` contract (`CONFIG_NAME=torchtitan_deepseek_v4_flash_12layer_ceiling_probe_bf16_4x8xmi350_canary.yaml`, `CANARY_TORCHTITAN_CONFIG=deepseek_v4_flash_12layer_ceiling_probe_bf16_canary`, `CANARY_GBS=128`). To reproduce the retained 4-layer baseline, override those two config variables back to the 4-layer target and train config. The launcher stages the selected local TorchTitan checkout, has a `check-config` action for a config-only runtime parse, refuses Primus runtime images, writes TorchTitan outputs to a run-scoped `DUMP_FOLDER`, supports `CANARY_RESUME_CHECKPOINT_PATH` for full-state resume from a checkpoint saved in another folder, supports checkpoint-free runs with `CANARY_CHECKPOINT_ENABLE=false`, and has a `pg-probe` action for direct-runtime FSDP/EP ProcessGroup identity. The default runtime is `onenexus/nexus-titan:rocm722-pytorch-nightly`, which has the PyTorch distributed APIs required by the current TorchTitan WIP snapshot. `CANARY_DEDICATED_EP_PG=true` is the default and sets `TORCHTITAN_DEVICE_MESH_EP_BACKEND_OVERRIDE=nccl`. `CANARY_LI_KL_BACKEND=triton_tl_dot` forwards `TORCHTITAN_DSV4_LI_KL_BACKEND=triton_tl_dot` for the opt-in Triton public-wrapper backend for deferred LI-KL backward. `CANARY_SHAREDKV_BACKWARD_MODE` and `CANARY_SHAREDKV_SCATTER_MODE` forward `TORCHTITAN_DSV4_SPARSE_MLA_SHAREDKV_BACKWARD_MODE` and `TORCHTITAN_DSV4_SPARSE_MLA_SHAREDKV_SCATTER_MODE`; keep shared-KV scatter unset for the safe auto default, and set `CANARY_SHAREDKV_SCATTER_MODE=triton_atomic` only to reproduce or harden the opt-in atomic path. `CANARY_MORI_AITER_BACKWARD_MODE=routed` forwards `TORCHTITAN_MORI_AITER_BACKWARD_MODE=routed` for the opt-in MORI-routed MoE backward integration gate. `CANARY_MORI_AITER_LOCAL_VJP_MODE=padded` forwards `TORCHTITAN_MORI_AITER_LOCAL_VJP_MODE=padded` to replace the local owned-expert autograd VJP with the padded owner-rank BMM VJP inside routed backward. `CANARY_MORI_AITER_OP_CACHE_MODE` forwards `TORCHTITAN_MORI_AITER_OP_CACHE_MODE`; the safe default is `backward`, which keeps forward MORI op caching while rebuilding backward MORI ops per call. Use `default` only to reproduce the cached-backward S4096 receive-count skew, or `none` to disable all MORI op caching. `CANARY_MORI_AITER_DX_RETURN_MODE=asyncll_staged` is diagnostic-only, and `CANARY_MORI_AITER_ASYNCLL_ALIGN_MODE=src_pos|route_key` selects the staged AsyncLL row-alignment gate; both long-sequence modes are currently rejected until shared route metadata exists. For the retained `8xMI350`, `S=4096`, `MBS=1`, `GBS=128`, `EP=8` target shape, unset `CANARY_CHUNKED_CE_NUM_CHUNKS` now auto-selects the measured CE2 candidate; other shapes keep TorchTitan's default 8 chunks unless explicitly overridden. `CANARY_MORI_SHMEM_HEAP_SIZE=32G` forwards `MORI_SHMEM_HEAP_SIZE=32G` and is required by the retained 8xMI350 `S=512`, `GBS=128` MORI routed run; the default MORI static heap OOMs at EP8 for that shape. `CANARY_PERSIST_JIT_CACHE=auto` bind-mounts image-keyed MORI/AITER JIT caches when MORI-backed paths are active, including routed MORI, native standard-EP compact collectives, and MORI/SDMA standard-EP hot-weight edges, under `${REMOTE_ROOT}/cache/jit/${CANARY_JIT_CACHE_KEY}/`; for those paths it also points C++ `MORI_KERNEL_DIR` at `/root/.mori/jit/gfx950_mlx5/latest` so runtime kernel autoload does not prefer stale build/install HSACO beside `libmori_ops.so`.
- Native compact note: when `CANARY_STANDARD_EP_COMPACT_COLLECTIVE_BACKEND=mori_native`, the launcher now defaults `CANARY_STANDARD_EP_MORI_NATIVE_COMPACT_ROLE_SPLIT=true` and `CANARY_MORI_SHMEM_HEAP_SIZE=32G` unless explicitly overridden. The 2026-06-10 heap A/B found `16G` too small for native compact handle construction and `48G` too large for FSDP headroom; this default is scoped to native compact so ordinary standard-EP or MORI-routed runs keep their previous heap behavior.
- `build_torchtitan_mori_overlay_image.sh` plus `Dockerfile.torchtitan_mori_overlay`: builds `onenexus/nexus-titan:rocm722-pytorch-nightly-mori` on `do-sonle5-mi350-gpu` by layering pinned `ROCm/mori` onto the internal NexusTitan TorchTitan image. Use this image with `CANARY_MOE_COMM_BACKEND=mori` and `CANARY_EXPERIMENTAL_MORI_AITER_MOE=1` for the guarded MORI dispatch + AITER fused-MoE bridge. A cold first run still compiles MORI/AITER kernels into the persistent cache; use a reuse run for timing evidence.

Older TileLang, Muon, and DeepSeek-V3 proxy launchers are intentionally not
part of this migrated branch. Keep exploratory probes and install checks outside
the repo until they become normal rerun paths.
